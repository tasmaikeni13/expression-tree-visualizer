Fundamentals

What is an Expression Tree?

An expression tree is a binary tree that represents a mathematical expression. Leaf nodes hold operands (numeric values or symbolic variables), and internal nodes hold operators (+, −, ×, /, ^). The tree's structure encodes evaluation order without needing parentheses.

🔢 Operand Nodes (Leaves)

Every leaf of the tree is an operand value. It may be a number such as 3 or a symbolic variable such as A or x. In the expressions ((3+4)/(9-2)) and (A+B), the leaves are the tokens that participate in each operation.

⚙️ Operator Nodes (Internal)

Every internal node is an operator. It combines the results of its left and right sub-trees. The root operator / divides the result of 3+4 by 9-2.

Expression tree for ((3+4)/(9-2)) — violet = operators, cyan = operands

🧪 Representative Expressions

The visualizer accepts fully parenthesised or precedence-driven infix expressions such as (A+B), x*(y+z), and ((a+4)/(b-2)) in addition to purely numeric forms.

Algorithm

How Infix → Postfix Conversion Works

The Shunting-Yard algorithm, invented by Edsger Dijkstra, converts human-readable infix notation into machine-friendly postfix (Reverse Polish Notation) using a single stack.

Algorithm Steps

  1. Read an operand → send it directly to the output queue.
  2. Read an operator → pop higher-precedence operators from the stack to the output, then push this operator.
  3. Read '(' → push it onto the stack as a barrier.
  4. Read ')' → pop operators to output until matching '(' is found; discard both parentheses.
  5. After all tokens: pop remaining operators from the stack to the output.

Building

Expression Tree Construction

Once we have the postfix sequence, building the tree is elegantly simple: scan left-to-right, pushing operand nodes and combining operator nodes.

📥 Operand Token

Create a leaf node with the operand value as its label. The operand may be numeric or symbolic. Push the node onto the stack with no children.

⚡ Operator Token

Create an operator node. Pop two nodes from the stack — the first becomes the right child, the second becomes the left child. Push the new sub-tree back onto the stack.

Symbolic example: (A+B) Postfix: A B + Prefix: + A B Tree: +(A,B)

Step-by-Step for 3 4 + 9 2 - /

Step 1: Read 3 → push Node(3) Stack: [3] Step 2: Read 4 → push Node(4) Stack: [3, 4] Step 3: Read + → pop 4,3 → Node(+) Stack: [+(3,4)] Step 4: Read 9 → push Node(9) Stack: [+(3,4), 9] Step 5: Read 2 → push Node(2) Stack: [+(3,4), 9, 2] Step 6: Read - → pop 2,9 → Node(-) Stack: [+(3,4), -(9,2)] Step 7: Read / → pop -,+ → Node(/) Stack: [/(+(3,4),-(9,2))] Root = /

Exploration

Tree Traversals

Three depth-first traversals produce three different views of the same expression — each corresponding to a classical notation.

🔀

Inorder

Left → Root → Right

Produces the original infix expression (with implicit parenthesisation).

3 + 4 / 9 - 2
⬇️

Preorder

Root → Left → Right

Produces prefix (Polish) notation — operators precede their operands.

/ + 3 4 - 9 2
⬆️

Postorder

Left → Right → Root

Produces postfix (Reverse-Polish) notation — used for evaluation.

3 4 + 9 2 - /

Computation

Expression Evaluation

The expression tree is evaluated bottom-up using postorder traversal: first compute the left sub-tree, then the right sub-tree, then apply the operator at the root.

Application

How the GUI Works

Launch the application and interact with mathematical expressions through a polished, dark-themed desktop interface.

1. Enter an Expression

Type an infix expression like ((3+4)/(9-2)), (A+B), or x*(y+z) into the input field. Real-time validation highlights errors as you type.

2. Build or Animate

Click ⚡ Build for instant results, or ▶ Animate to watch the Shunting-Yard algorithm and tree construction unfold step by step.

3. Explore Traversals

Click Inorder, Preorder, or Postorder — nodes highlight in real time following the traversal path, with educational descriptions.

4. View Evaluation

Numeric expressions show each sub-expression computed bottom-up with a calculation log and final result. Symbolic expressions remain available for conversion, traversal, and tree exploration.

5. Interact with the Tree

Zoom in/out with the scroll wheel, drag to pan, hover on nodes for tooltips showing whether each node is an operator or operand.

6. Export Your Work

Save the tree as a PNG image or export the explanation log as a text file for study notes or inclusion in reports.
